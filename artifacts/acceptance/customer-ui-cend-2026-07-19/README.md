## 验收标准

- 页面宽度与视口一致，无横向滚动或内容裁切。
- 移动端交互目标不小于 `44 × 44px`。
- 底部导航与主要 CTA 不互相遮挡。
- 错误状态可恢复，成功状态仅在服务端确认后展示。
- 顾客可见文案中文化，不展示内部工作流或调试信息。
- 使用项目现有 Phosphor 图标与 `@xlb/ui` 组件，不引入伪造素材。

## 复现方式

在项目根目录执行：

```powershell
pnpm --filter @xlb/customer exec vite --host 127.0.0.1 --port 5173 --strictPort
node scripts/customer-ui-visual-qa.mjs
```

自动验收脚本会使用稳定的本地 API 响应，不依赖生产环境或真实用户数据。
