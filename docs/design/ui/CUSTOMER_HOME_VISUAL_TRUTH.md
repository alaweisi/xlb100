## 视觉构成契约

1. 顶部是“喜乐帮”品牌、服务承诺与通知入口。
2. 城市/区域选择和全量服务搜索合并为一条高圆角功能栏。
3. “全部服务”按 4 列展示正式目录中的 16 个一级类目；名称、顺序和可用性来自正式目录/API，不从图片硬编码。
4. 每个类目使用独立、语义匹配的暖白陶瓷质感 3D 图像，主色为墨绿，必要时用少量陶橙提示；不得复用同一个扳手或使用 Emoji 占位。
5. “推荐服务”使用横向卡片流，图像、标题、标签与动作层级遵循真相图；推荐事实必须来自权威接口，接口未提供时使用诚实空态或隐藏该区块。
6. “附近师傅”使用横向师傅卡，认证、距离、可接单状态等均来自权威接口；图片中的示例人物和文案不是业务事实。
7. 信任保障条位于主内容与底部导航之间，表达实名认证、价格透明、服务留痕和售后保障。
8. 一级导航固定为 `首页 / 客服 / 新报修（中心 +） / 订单 / 我的`；通知通过顶部铃铛进入，不占一级导航。
9. 奶油白页面、墨绿标题、深灰正文、明亮陶橙主动作、柔和白色表面与克制玻璃边缘构成顾客端基础视觉语言。

## 真相边界

- 视觉真相：本文和唯一 PNG 决定构图、层级、色彩关系、材质、圆角、图像语言与导航表达。
- 业务真相：`docs/catalog/OFFICIAL_SERVICE_CATALOG_SOURCE.md`、后端流程、权限、`@xlb/api-client`、共享类型和校验器决定类目、价格、推荐、师傅、状态与动作。
- 实现真相：当前代码、通过的测试和同视口运行截图决定“是否已经实现”，不得反向修改视觉真相。
- 图片中的示例服务、推荐、师傅、城市和可接单状态只表达版式，不授权虚构业务数据。
