## 硬性要求

- [ ] 用户可见界面全部中文；
- [ ] 使用三端真实 App 页面，不创建静态 Demo；
- [ ] 每个业务事实来自后端契约；
- [ ] 输入、确认、提交中、成功、失败、冲突、重复、离线和结果未知均按适用范围实现；
- [ ] 操作结果持久显示，不用 Toast 代替业务结果；
- [ ] 跨端交接显示下一责任人和入口；
- [ ] Edge 完成真实操作与截图；
- [ ] 总账更新到实际达到的状态；
- [ ] 四道门禁运行并报告。

## 停止条件

- 缺少正式服务类目；
- 缺少金额、退款、权限或责任规则；
- 后端没有对应状态/API；
- 需要生产数据、真实 Provider、push 或 deploy 授权；
- 新增范围与现有 Slice Contract 冲突。
