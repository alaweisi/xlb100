# 喜乐帮前端产品边界

本文件是当前 UI 生产总控的产品边界，不得把三个不同产品合并成一个“后台”。

| 产品 | 当前定位 | 主要设备 | 当前工程范围 |
| --- | --- | --- | --- |
| 顾客端 | 顾客下单与服务过程 App | 手机 | `apps/customer` |
| 师傅端 | 师傅接单、履约与收益 App | 手机 | `apps/worker` |
| 运营端 | 城市即时运营、派单、客服、审核和异常处置 App | 手机 | `apps/admin` |
| OA | 专业组织管理、复杂配置、批量处理与长表单系统 | 桌面网页 | 后续独立产品，不计入当前三端竣工 |
| 数据大屏 | 实时指标、告警和态势展示 | 大屏 | 后续独立产品，不计入当前三端竣工 |

## 当前运营端约束

- `apps/admin` 必须以 390×844 手机视口作为主验收基线，不能恢复桌面侧栏式网页框架。
- 主导航为五项底部导航；完整业务入口由角色化工具箱承载。
- 身份、城市、角色和业务权限使用全屏手机 Gate；无权限入口不得诱导用户反复撞门禁。
- 即时运营能力留在手机运营 App；适合批量、复杂配置和长时办公的能力在页面中标注未来 OA 承接边界，但不得删除现有真实业务 API。
- 当前总账中的 `admin` 是历史技术终端键，产品展示名称统一为“运营端”。

## 验收边界

当前三端 UI 竣工只统计顾客端、师傅端和运营端。OA 与数据大屏必须在后续独立立项、建立自己的切片总账、Frame Map、证据目录和上线门禁，不能用当前运营 App 的证据冒充完成。
