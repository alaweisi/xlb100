# Phase 15.0D Component Render Rules

These rules deepen the Figma snapshot into implementation guidance. They do not authorize free redesign and do not require code changes in Phase 15.0D.

## Cross-Component Rules

- Role color must be token-driven, not copied as one-off hex values across pages.
- Loading, empty, error, and retry states must be honest reflections of API state.
- Components must reserve dimensions for icons, labels, and loading text to avoid layout shift.
- Figma 390px mobile frames are source of hierarchy; desktop admin may adapt density but not invent unrelated modules.
- Missing components should be added only when a Phase 15 page requires them.
