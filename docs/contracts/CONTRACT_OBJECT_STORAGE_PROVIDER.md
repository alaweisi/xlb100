# Object Storage Provider Contract

Phase: 18

## Provider Boundary

`ObjectStorageProviderEnvelope` supports exactly two providers:

- `local`: private filesystem bytes with status `stored_local`
- `mock`: process-memory bytes for tests with status `stored_mock`

Every successful envelope must state:

- `externalProviderExecuted: false`
- `publicUrl: null`
- provider name `xlb-local-filesystem` or `xlb-memory-mock`
- private `xlb-local://` or `xlb-mock://` storage URI
- SHA-256 checksum, byte size, MIME type, object key, and stored timestamp

An unsupported `XLB_OBJECT_STORAGE_PROVIDER` value fails application startup. There is no Alibaba OSS, S3, COS, Qiniu, or other cloud-provider client in Phase 18.

## Object Safety

- Object keys are generated by the backend and restricted to lowercase scoped path segments plus `.jpg`, `.png`, or `.webp`.
- Local paths are resolved beneath the configured private root and cannot traverse outside it.
- Local writes use exclusive creation and do not silently overwrite evidence.
- Object content is exposed only through an authenticated API with `Cache-Control: private, no-store` and `X-Content-Type-Options: nosniff`.

## Upload Safety

- Maximum size: 5 MiB.
- Allowed declared MIME types: `image/jpeg`, `image/png`, `image/webp`.
- Declared MIME, filename extension, and binary signature must agree.
- Empty files, unsafe names, path separators, control characters, and double-dot names are rejected.
- Phase 18 validates signatures but does not claim malware scanning; the persisted state is explicitly `not_malware_scanned_local`.
